PROPOSAL:

OpenClaw is a config-first agent OS. Gateway control plane (Node.js WebSocket on 127.0.0.1:18789) routes everything. SOUL.md defines personality — no code needed. Plugin system uses TypeScript modules with OpenClawPluginApi for providers, tools, hooks, channels. Skills are Markdown capability definitions injected into system prompts. Sessions isolated via JSONL append-only files with scoping models (main/per-peer/per-channel-peer). Session keys ARE addresses: pipeline:project:role enables direct peer-to-peer messaging. Lobster workflow engine provides deterministic YAML workflows with sub-workflow loops and JSON data flow. Factory line decomposition: each agent gets own tools, memory, identity, workspace; Programmer (exec+write, Opus), Reviewer (read-only, Sonnet), Tester (exec+runners, Sonnet). Local-first with all state in JSONL/JSON. Scale: 30 agents, 5B tokens/day, 1.5M LOC refactored in 7 days.

Xenota is a persistent sentient agent infrastructure. Nucleus provides persistent identity: OODA cognitive loop, genome (64 genes/8 modules), mind state (genome/imprints/impulses), tick-bounded work windows. Projections are containerized external capability surfaces (Chat, GitHub Contributor, MCP) — gateway-mediated, typed. Cortex handles projection lifecycle with authority split (nucleus decides, cortex executes). Repertoire system has packaged LLM routines with contract.yaml, multi-backend variants, eval system. Chaperone console is privileged human interface. BAR gate is mandatory post-awakening safety checkpoint. Membrane does 5-step inbound validation with quarantine. XSM is deterministic swarm monitor (5 signal channels, 9 states, wrangle engine). Gas Town handles multi-agent orchestration (Mayor/Crew/Polecats/Witness/Refinery, beads, mail, hooks). ACP delegation supports external agents including OpenClaw transport.

KEY DIFFERENCES:
- Scale focus: OpenClaw=throughput (30 agents, 5B tokens) vs Xenota=identity/safety
- Workflow: OpenClaw=Lobster YAML deterministic vs Xenota=molecules/formulas
- Extension: OpenClaw=plugins(TS)+skills(MD) lightweight vs Xenota=repertoires heavy
- State: OpenClaw=JSONL simple vs Xenota=SQLite+Dolt complex

Focus on: What is the fastest path for Xenota to reach OpenClaw-level multi-agent throughput? What can be cut or simplified? What OpenClaw patterns ship fastest with highest impact? What is Xenota over-engineering that OpenClaw solved simply?
